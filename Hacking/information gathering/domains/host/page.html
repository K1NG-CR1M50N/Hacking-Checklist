The host command does 2 thing: Forward Lookup for DNS and Reverse Lookup for IP
You can obtain the IP or a Domain if you have the other one!


DNS Request (you can choose the request with -t option):
  • SOA (State of Authority) [RFC 1035 and 2308]: Ask for some info about a DNS Zone
  • NS (Name Server) [RFC 1035]: Ask for Authoritative Servers on a defined DNS Zone
  • A [RFC 1035]: Record Address for the IPv4
  • AAAA [RFC 3596]: Record Address for the IPv6
  • MX (Mail Exchange) [1035]: Ask the DNS about which server is in charge to receive and handle emails and where the emails are routed
  • CNAME (Canonical Name) [RFC 1035]: Ask for some Domain aliases
  • CAA (Certificate Authority Authorization) [RFC 6844]:
  • PTR (Pointer Record) [RFC 1035]: Pointer to a canonical name, it is used for reverse mapping
  • TXT (Text Record) [RFC 1035] : Record with some kind of human-readble information, commony used to add more info about a DNS


Zone Transfer (ZT): Mechanism used for replicate the DB from a Master Name Server to a Slave Name Server

How to make a zone transfer:
  • Use host  on a domain and you will get the DNS
  • Use the -l option with the domain and the DNS: you will get a zone transfer!
